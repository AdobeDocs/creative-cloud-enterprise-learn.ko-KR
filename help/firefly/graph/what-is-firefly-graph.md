# &#x200B;1. [!DNL Firefly Graph]이란 무엇입니까?

대부분의 생성형 인공지능 도구는 한 가지 프롬프트에서 한 가지 출력을 제공합니다. 만약 약간이 바뀌면 손으로 전체를 재건하고 최종 파일만 넘길 게 없어

Firefly 그래프는 다르게 작동합니다. 단일 프롬프트 대신 모든 입력, 변환 및 출력이 함께 캡처되는 시각적 단계별 작업 과정인 **그래프**&#x200B;을 만듭니다. 한 단계를 변경하고 다시 실행합니다. 전체 체인을 다시 작성하지는 않습니다. 모든 단계는 팀이 검사하고, 조정하고, 그대로 전달할 수 있는 눈에 보이는 노드입니다.

![시각적 그래프 스크린샷](../assets/what-is-1.png)

즉, Graph는 창의적인 프로세스를 대체하지 않습니다. 이 프로세스를 보고, 재사용하고, 크기를 조절할 수 있는 프로세스로 전환합니다.

## 단계, 표시됨

오늘 이미 실행한 워크플로우를 생각해 보십시오. 간단하게 시작하고, 입력을 수집하고, 변형을 만들고, 다듬고, 내보낼 수 있습니다. 그래프는 이러한 단계를 변경하지 않으며 각 단계를 표시하고 다시 사용할 수 있게 합니다.

| 지금 내 워크플로우 | 그래프의 표시 여부 |
|---|---|
| 간단하게 시작하기 | 브리프가 그래프 입력이 됩니다. |
| 입력 수집 | 입력이 노드에 직접 연결됨 |
| 다양한 효과 적용 | 변형은 병렬로 실행됩니다. |
| 다듬기 | 나머지 부분을 터치하지 않고 한 노드 다듬기 |
| 내보내기 | 그래프에서 직선 내보내기 |

바로 이런 변화입니다. 같은 작업이죠. 하지만 모든 결정은 이제 여러분이 보고, 조정하고, 재사용할 수 있는 것입니다. 다음에 처음부터 다시 써야 할 블랙박스가 아니라 말이죠.
